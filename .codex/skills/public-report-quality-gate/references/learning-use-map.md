# Learning Use Map

Use this before drafting or claiming review-ready status when a reader-facing
output depends on prior learning artifacts, article/repo/example references, or
the phrase "use what we learned".

## Required Fields

| Field | Required Meaning |
|---|---|
| Learning artifact | Wiki path, source note, example, or gate review artifact |
| Domain / topic | What the source is about |
| Capability type | `content`, `structure`, `visual`, `evidence`, or `boundary` |
| Learning point | The reusable lesson in one sentence |
| Output use | Section, visual, metric, evidence block, boundary statement, or skip |
| Skip reason | Required when relevant learning is not used |

## Capability Types

| Type | Output effect |
|---|---|
| `content` | Shapes claims, concepts, vocabulary, or arguments |
| `structure` | Shapes article order, section rhythm, page split, or reading map |
| `visual` | Shapes lifecycle diagrams, comparison visuals, visual density, or figure roles |
| `evidence` | Shapes metric choice, cases, observable scenes, or proof style |
| `boundary` | Shapes future/current separation, non-claims, or adoption limits |

## Rules

- Do not call capability types "domains".
- Do not claim learning was used unless a concrete output element is named.
- Do not force every learning point into the output; skip with a reason.
- Do not use learned evidence style to invent local metrics.
- Do not promote repeated learning use into rules without the promotion path.
